# reDirector
reConnect's reverse proxy for reMarkable tablets.

reDirector runs on-device and forwards reMarkable traffic to a custom upstream server.

## Installation
Pick the installer that matches your device architecture.

### reMarkable 1 / 2 (32-bit)
1. Connect to the tablet over SSH ([guide](https://developer.remarkable.com/documentation/developer-mode)).
2. Identify your tablet's IP address and connect:
   ```bash
   ssh root@<tablet-ip>
   ```
3. Download installer:
   ```bash
   wget https://github.com/Alexandre2006/reDirector/releases/download/v1.0.0/installer-32
   ```
4. Make it executable and install:
   ```bash
   chmod +x installer-32
   ./installer-32 install
   ```
5. Enter your upstream address when prompted.

### reMarkable Paper Pro / Pro Move (64-bit)
1. Enable [Developer Mode](https://developer.remarkable.com/documentation/developer-mode).
2. Connect to the tablet over SSH ([guide](https://developer.remarkable.com/documentation/developer-mode)):
   ```bash
   ssh root@<tablet-ip>
   ```
3. Download installer:
   ```bash
   wget https://github.com/Alexandre2006/reDirector/releases/download/v1.0.0/installer-64
   ```
4. Make it executable and install:
   ```bash
   chmod +x installer-64
   ./installer-64 install
   ```
5. Enter your upstream address when prompted.

## Uninstall
Run the same installer used for installation with:

```bash
./installer-XX uninstall
```

Replace `XX` with `32` or `64`.

## Repair
If a tablet software update leaves reDirector in a broken state, rerun the installer with:

```bash
./installer-XX repair
```

Replace `XX` with `32` or `64`.

## Configuration
Configuration file:

```text
/home/root/reDirector/config.toml
```

### `certificatepath` / `certificatekeypath`
Path to the proxy certificate and private key.

These are generated automatically during installation and usually should not be changed.

### `clientcertificatepath` / `clientcertificatekeypath`
Path to optional client certificate and key used when authenticating to upstream.

These are not generated by default.

### `listenaddress`
Address/interface reDirector listens on.

Default: `:443`.

### `upstreamaddress`
Address of the upstream server reDirector forwards requests to.

Set during installation.

## Troubleshooting
- If the device was recently updated, run `repair`.
